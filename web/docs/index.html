IntroductionMenu

SixOrNot is a Firefox extension intended to make it easy for you to identify when the website you're connecting to supports the current generation Internet Protocol (version 6). It does this by checking the site's DNS records and displaying an icon either in the address bar or as a button (which you may place anywhere buttons can usually be placed in Firefox). This icon indicates whether the site currently supports IPv6 (or only the legacy Internet Protocol, IPv4).

InstallationMenu

Installation is via the usual Firefox method, simply download the .xpi format attachment and accept when prompted to install. SixOrNot is a "restartless" or "bootstrapped" extension - this means you don't need to restart Firefox when it is installed and you should see the addressbar icon appear immediately upon installation.

Upgrading should be similarly easy and handled automatically by the addons interface.

InterfaceMenu

The SixOrNot interface is fairly simple. An icon is displayed either in the address bar near the star icon or as a button. By default the address bar icon is enabled and the button is placed into the button palette which you can access by the standard UI customisation method. Whether accessed through the button or address bar icon the functionality is identical. On mouseover the icon displays a tooltip which shows information about the site you are connecting to. The IPv6/IPv4 addresses of the site are shown in the Remote section along with its domain name. Additionally the IPv6/IPv4 addresses of your own client are shown under the Local section.

Clicking on either the address bar icon or the button will open a popup menu. This menu allows you to copy IP address information to the clipboard. To copy all the IP addresses for a particular domain click the first entry in the menu (which will show the domain name of the remote site). This will copy the domain name followed by the IP addresses in comma separated format. If you wish to copy an individual IP address select it from the list and only that address will be copied to the clipboard.

The same functionality is available for the local host entries. Below the IP address section are options to configure settings (see the Configurable Settings section below for more details on what these do) and a link to the SixOrNot website where this documentation can be found.

Icon MeaningsInterface

The IP addressing state of the remote website is displayed via a set of 5 icons, their meanings are outlined below. Note that there are two sets of icons to choose between (one colourful, the other greyscale). The grey icons look nicer when used with the button on Max OSX for example, and can be better for those who have difficulty distinguishing between colours.

  • This icon can mean a variety of things, generally it indicates that no IP address information exists for the current location or that there was a failure obtaining it. Try navigating to a website and ensuring you are not using a Proxy and that Offline mode isn't enabled.
  • This icon indicates that only AAAA (IPv6) records were found for the current domain. This generally means that the domain in question only supports IPv6. For example: ipv6.entropy.me.uk.
  • This icon indicates that both AAAA (IPv6) and A (IPv4) records were found for the current domain and SixOrNot detected that the local host has at least one valid IPv6 address. You are probably connecting to this website using IPv6, the current generation of the Internet Protocol. For example: entropy.me.uk.
  • This icon indicates that both AAAA (IPv6) and A (IPv4) records were found for the current domain but SixOrNot has not detected any local valid IPv6 addresses. While the remote domain does support IPv6 you are probably not able to connect to this site using native IPv6.
  • This icon indicates that only A (IPv4) records were returned for the current domain. Either your DNS is not capable of returning AAAA records or (more likely) the domain in question supports only the legacy version of the Internet Protocol suite. Why not ask why they haven't upgraded yet? An example can be found here: ipv4.entropy.me.uk

Configurable SettingsInterface

SixOrNot allows you to configure two settings.

  1. Whether or not the address bar icon should be displayed. This is either on or off. This can be configured via the menu either by clicking on the address bar icon or the button. If you turn the address bar icon off and don't see the button try opening the button palette via the standard right-click+"Customise" functionality in Firefox. You can drag the address bar icon from there to anywhere you like on the UI. Note: If you want the address bar icon back you need to switch it on via the button's menu. You can always remove the button again afterwards by dragging it back to the button palette.
  2. Whether to use the colourful icon set or the greyscale one. This can be switched via either menu at any time. The grey icons don't look quite as good except when being used on Mac OSX with the button (where most buttons are greyscale). They are also good for those with specific accessibility needs where colours aren't a sufficient means of communicating information.

These two options can also be configured via the following two boolean values in "about:config":

extensions.sixornot.showaddressicon
extensions.sixornot.use_greyscale

The other two sixornot settings control the location of the button and should not be changed manually.

IP Address ConceptsMenu

IP addresses are fundamental to the operation of the Internet, billions of people use millions of them every day and yet they are invisible in day-to-day use. Usually we don't need to care about IP addresses and in most cases one only becomes aware of them when something is broken. This is due to a system called DNS - the Domain Name System. This masks the technical complexity of addressing behind easy-to-remember domain names. When you visit a website behind the scenes the domain name of that website (everything between the "http://" and the first "/") is looked up via a DNS query to produce an IP address. These addresses are then used to determine how to communicate with the server responsible for serving that website.

There are currently two flavours of IP address around because the Internet is going through a migration period from the old way of addressing things (known as Internet Protocol version 4, or IPv4) to a new addressing system (known as Internet Protocol version 6, or IPv6). The main difference is the number of addresses available, though this is by far the most important difference. IPv4 addresses are represented as 32-bit numbers, the largest number which can be represented in this way is 2^32 (2 to the power of 32) or 4,294,967,296. That sounds like a lot but we are rapidly running out of them (and in practice we can't even use all of those). IPv6 addresses are represented by 128-bit numbers, the largest number which can be represented in this way is 2^128 (2 to the power of 128) or 340,282,366,920,938,463,463,374,607,431,768,211,456 (that's approximately 340 billion billion billion billion, or 3.4x10^38).

So why is there a problem? Adoption of the current IP addressing standard (IPv6) has been slow. Most traffic on the Internet still uses the legacy addressing scheme. At the time of writing less than 1% of all Internet traffic uses IPv6. Given the impending exhaustion of IPv4 addresses (at the time of writing the Asia-Pacific regional registry, APNIC, has already run out) this is a major problem facing the Internet community. The transition from IPv4 to IPv6 isn't an easy one and involves a lot of work, this is typically work which goes unnoticed to the end user. This is as it should be and for the vast majority of people IP addressing will continue to remain an invisible part of their Internet-using experience. The flip side of this invisibility however is that it's easy to overlook the pioneers who are already using IPv6 and raise awareness among those that are not. I am hopeful that by exposing the IP version being used as part of your web browsing this extension will help make it just a little bit more visible.
